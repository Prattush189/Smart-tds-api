## Conversion rules locked (also apply to the BAL query port in Phase 3)
- **Identifiers lowercased.** PG folds unquoted names to lowercase; Npgsql `reader["MixedCase"]` is case-insensitive ⇒ existing C# mapping and bracket-stripped queries keep working **without per-identifier quoting**.
- `int IDENTITY(1,1)` → `integer GENERATED BY DEFAULT AS IDENTITY` (BY DEFAULT so existing PKs load; reset sequence after).
- `nvarchar(n)`→`varchar(n)`, `nvarchar(max)`→`text`, `bit`→`boolean`, `datetime`→`timestamp`,
  `decimal/numeric(p,s)`→`numeric(p,s)` (**exact** — money is numeric here, NOT float), `image/varbinary`→`bytea`, `float`→`double precision`.
- `getdate()`→`now()`; bit defaults `0/1`→`false/true`.
- **Reserved-word columns that MUST be quoted everywhere** (incl. BAL queries): **`"limit"`** (`tdsentriessection`, `tdsrate`) and **`"desc"`** (`f15hn`, per-year DB). All other mixed-bag names (`name,value,mode,date,number,period,state,status,section`) are non-reserved in PG.
- `SCOPE_IDENTITY()` (used pervasively in BAL inserts) → `INSERT … RETURNING id`.
- ⚠️ PG16 ships an `integer→boolean` cast but it's **explicit-only** — bare `VALUES (1)` into a boolean fails. Seed values are converted to `true/false` in `convert_seed.ps1` (quote/paren-aware tokenizer).

## How to rebuild (repeatable)
```powershell
# requires Docker; one-time ephemeral server:
docker run -d --name sttest -e POSTGRES_PASSWORD=pw -p 55432:5432 postgres:16
# build both databases + verify counts:
_migration\phase1\run_pg_migration.ps1 -Container sttest -Year 26
```

## What remains to FULLY close Phase 1 for production
The schema (both DBs) and the **master reference data** are migrated and validated. The only piece not yet done is bulk migration of **live transactional firm data** (real `assessee`, `payee`, `tdsentry`, `salary`, … rows), because the provided scripts contain schema + reference data only — no per-year transactional data dump.

To finish that, one of:
1. A data-only export of live `SmartTds26` (+ any in-use `MasterDbTds` firm rows) → run through the same converter pattern, then reconcile with row-count + checksum per table; **or**
2. Point a small ETL (Npgsql `COPY`) at the live SQL Server and stream tables across.

The pattern is proven (converter + load + reconcile). This dependency is data-availability, not technical risk.
